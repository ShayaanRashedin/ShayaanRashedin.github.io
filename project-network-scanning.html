Network Scanning & Reconnaissance

Coming soon — this page is ready for your full write-up.

Executive Summary

In an isolated VirtualBox lab, I enumerated a Metasploitable2 target from Kali. I validated reachability, ran full-port and service/version scans with Nmap, grabbed HTTP banners, and executed a GVM/OpenVAS assessment. The target exposed 30+ TCP services including legacy daemons (rsh/rexec), weak/default credentials (MySQL, PostgreSQL, VNC), and remote-code-execution paths (vsftpd 2.3.4 backdoor, DistCC, Tomcat AJP “Ghostcat”), consistent with a deliberately vulnerable host.

Lab Topology

  • Attacker: Kali Linux (VirtualBox) – interfaces eth0/eth1
  • Target: Metasploitable2 at 192.168.56.101 (Host-Only)
  • Safety: Performed entirely in a closed home lab.

Key Commands

# Discover target & verify reachability
ping 192.168.56.101

# Full TCP scan (all ports)
nmap -p- -T4 192.168.56.101 -oN scans/full-port-scan.txt

# Service/version + default scripts + OS guess
nmap -sV -sC -O 192.168.56.101 -oN scans/version-os-scan.txt

# HTTP banner grab (evidence of Apache/PHP)
curl -I http://192.168.56.101:80/

Nmap Highlights

  • FTP 21/tcp – vsftpd 2.3.4 (backdoor build), also shell on 6200/tcp
  • SSH 22/tcp – OpenSSH 4.7p1
  • Telnet 23/tcp – Linux telnetd
  • SMTP 25/tcp – Postfix smtpd
  • DNS 53/tcp – ISC BIND 9.4.2
  • HTTP 80/tcp – Apache/2.2.8 (Ubuntu), PHP/5.2.4
  • SMB 139/445 – Samba 3.X–4.X
  • rlogin/rsh/rexec 513/514/512 – legacy remote shells
  • FTP (alt) 2121/tcp – ProFTPD 1.3.1
  • MySQL 3306/tcp – 5.0.51a; PostgreSQL 5432/tcp – 8.3.x
  • DistCC 3632/tcp – remote compilation service
  • VNC 5900/tcp – Protocol 3.3
  • AJP 8009/tcp – Tomcat; HTTP alt 8180/tcp – Tomcat/Coyote JSP

Findings & Remediation (top criticals)

ServiceVersionIssue / CVERiskRecommended Action
FTP 21/tcpvsftpd 2.3.4 Backdoored build opens shell on 6200/tcp (CVE-2011-2523) HighRemove/upgrade vsftpd; verify no rogue 6200 listener.
MySQL 3306/tcp5.0.51a Default/empty root credential detected HighSet strong root password; restrict to localhost/VPN; upgrade.
PostgreSQL 5432/tcp8.3.1 Default creds postgres:postgres; TLS weakness (CCS Injection CVE-2014-0224) HighRotate credentials; enforce TLS ≥1.2; patch/upgrade.
DistCC 3632/tcp— Unauth RCE (CVE-2004-2687) HighDisable service or restrict by ACL; patch.
Tomcat AJP 8009/tcp5.5 (AJP) “Ghostcat” LFI/RCE (CVE-2020-1938) HighDisable AJP or upgrade Tomcat & require secrets.
HTTP 80/tcpApache 2.2.8 + PHP 5.2.4 CGI arg injection RCE (CVE-2012-1823) + many PHP vulns; PUT/DELETE enabled HighUpgrade PHP/Apache; disable dangerous HTTP methods; limit DAV.
VNC 5900/tcpProtocol 3.3 Weak password “password” accepted HighSet strong VNC auth; tunnel via SSH/VPN; consider removal.
rsh/rexec 514/512netkit Clear-text remote shells exposed HighDisable r-services; use SSH only.
OSUbuntu 8.04 End-of-life platform (no security updates) HighMigrate to a supported LTS; re-baseline hardening.

Evidence for the items above comes from the OpenVAS/GVM report (19 High, 39 Medium, 4 Low on 192.168.56.101), including vsftpd backdoor (21/6200), default DB credentials (3306/5432), DistCC RCE (3632), Tomcat Ghostcat (8009), PHP CGI RCE and outdated PHP on 80, weak VNC password (5900), r-services exposure (512/514), and OS EOL.

Next Steps

  • Remove/disable unused legacy services; enforce MFA and password rotation.
  • Patch Tomcat/PHP/DBs; restrict DB ports to localhost or VPN subnets.
  • Harden web (disable PUT/DELETE, least-privileged runtime); add IDS/centralized logs.
  • Re-scan to verify remediation and track deltas.
