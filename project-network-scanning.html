Network Scanning & Reconnaissance — Nmap & OpenVAS

Tools: Kali Linux, Nmap, OpenVAS / GVM, VirtualBox (host-only network)

Objective: Identify exposed hosts and services in a controlled lab network and produce prioritized findings for follow-up exploitation or defensive hardening.

Environment & Setup

Lab runs in VirtualBox with the attacker (Kali) and target (Metasploitable2) on a host-only network (example: 192.168.56.0/24). Always snapshot target VMs before running noisy scans. The images and outputs below were captured from the Kali attacker terminal and the Metasploitable2 target.

Reconnaissance Workflow (phases)

The work follows three phases: Host discovery, Port/Service enumeration, and Automated vulnerability scanning. Each phase contains the commands used and supporting screenshots.

Phase A — Host discovery

Commands (Host discovery) 
# Basic ICMP check to confirm the target is reachable
ping 192.168.56.101

Why: confirms host is up and reachable before scanning. (See Figure 1)

Phase B — Full port discovery (TCP full-range)

Commands (Full TCP port scan) 
# Full TCP SYN scan across all ports
nmap -sS -p- -oN scans/full-port-scan.txt 192.168.56.101

Why: finds every open TCP port on the host so you know which services to enumerate further. (See Figures 2 & 3)

Phase C — Service & version enumeration

Commands (Service/version detection) 
# Service/version detection with default scripts for common ports
nmap -sV -sC -oN scans/version-os-scan.txt 192.168.56.101

Why: identifies service names and versions (e.g., vsftpd 2.3.4, Apache 2.2.8), which maps to CVEs and exploitability. (See Figures 4 & 5)

Auxiliary host verification

Commands (local host verification) 
# Verify HTTP response headers quickly
curl -I http://192.168.56.101:80

# Check target's network interfaces (on the target)
ifconfig
ip addr

Why: the HTTP headers confirm server technology (Apache + PHP version), and the target's ifconfig/ip output confirms the VM IPs and interfaces used during the lab. (See Figures 6–8)

Analysis & Findings

Combined manual Nmap enumeration and OpenVAS outputs revealed multiple high-risk services and default/weak configurations typical of Metasploitable2. These include remote code execution/backdoor issues (vsftpd, distcc), default credentials (MySQL, PostgreSQL), and web stack vulnerabilities (PHP/TWiki/Tomcat). Prioritization used CVSS and exploit availability.

Remediation & Defensive View

  • Patch or remove obsolete services; upgrade PHP/Apache/tomcat to vendor-fixed releases.
  • Disable network-exposed services not required by the host (e.g., rexec, rsh, old FTP if not used).
  • Enforce strong credentials and rotate default passwords immediately.
  • Network segmentation & host-based firewalls to limit lateral movement.
  • Feed scan results into SIEM (Splunk/Wazuh/ELK) and build alerts for unusual service discovery or high CVSS findings.

Reflection

This recon phase taught me how to combine targeted Nmap scans with automated scanners to form a prioritized attack surface map — a repeatable process useful for red- and blue-team workflows. Skills reinforced: OS/service fingerprinting, CVE mapping, safe lab scanning practices, and evidence capture for reporting.
