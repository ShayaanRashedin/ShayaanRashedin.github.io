Network Scanning & Reconnaissance

Mapping exposed services, enumerating versions, and assessing vulnerabilities in a controlled lab (no exploitation performed).

Tools: Nmap, curl, OpenVAS/GVM Target: Metasploitable2 Attacker: Kali Linux Scope: Discovery → Enumeration → Vulnerability Assessment

Project Overview

Objective: identify all externally reachable services on an intentionally vulnerable host, fingerprint versions, and assess potential vulnerabilities without exploiting them. The outcome is a prioritized service map and an OpenVAS-backed risk snapshot that informs remediation and follow-up testing.

  • Discovery methods: ICMP/ARP sweep, full TCP scan
  • Enumeration: -sV service banners, -O OS guess, HTTP headers
  • Reporting: OpenVAS XML → themed HTML

Discovery & Reconnaissance

The reconnaissance methodology proceeds from least to most intrusive: confirm reachability, enumerate ports, then fingerprint services and OS. Each phase includes the exact commands and rationale.

Phase A — Host Discovery

Simple ICMP reachability check to validate L3 path before scanning.

Commands used — Host discovery 
ping 192.168.56.101
# Rationale: quick L3 reachability check before deeper scans.

Phase B — Full TCP Port Discovery

A SYN scan across the full TCP range identifies all listening services to scope subsequent enumeration. Timing template kept modest for accuracy on a local VM network.

Commands used — Full TCP scan 
nmap -sS -p- -T4 -oN nmap-full.txt 192.168.56.101
# -sS: SYN scan
# -p-: all 65535 TCP ports
# -T4: faster timing (safe on local lab)
# -oN: save normal output for evidence

Phase C — Service & OS Enumeration

Service version detection provides banner strings that map to known vulnerabilities; OS fingerprinting validates platform assumptions for later remediation planning.

Commands used — Service & OS enumeration 
nmap -sV -sC -O -oN nmap-services-os.txt 192.168.56.101
# -sV: service/version detection
# -sC: default safe scripts (NSE)
# -O : OS fingerprinting
# -oN: save output for reporting

Phase D — Application Headers

HTTP headers provide fast confirmation of web stack components (server/PHP). This is useful to corroborate Nmap results and quickly check for end-of-life versions.

Command used — HTTP headers 
curl -I http://192.168.56.101
# Rationale: non-invasive banner grab to confirm web stack versions.

Vulnerability Scanning & Reporting

Manual enumeration was complemented with an OpenVAS “Full and Fast” policy scan to prioritize risks and attach CVE references. The scanner was executed locally in the lab; only the target host was in scope.

Commands & workflow — OpenVAS / GVM 
# Start Greenbone stack (varies by install)
gvm-start

Summary of Findings

IP Port Service / Version Severity CVE Notes
192.168.56.102 21/tcp vsftpd 2.3.4 High CVE-2011-2523 Backdoored release; unauthenticated shell possible (referenced only; no exploitation performed).
192.168.56.102 6667/tcp UnrealIRCd 3.2.8.1 High CVE-2010-2075 Malicious backdoor in distributed tarball (known lab finding).
192.168.56.102 3632/tcp distcc High CVE-2004-2687 Remote command execution exposure; mitigation is to disable or restrict to trusted build hosts.
192.168.56.102 3306/tcp MySQL 5.0.51a High CVE-2009-2446 Outdated DB server; commonly weak defaults in this image.
192.168.56.102 23/tcp Telnet Medium — Legacy plaintext protocol; disable and use SSH with key-based auth.
192.168.56.102 80/tcp Apache httpd 2.2.8 (PHP 5.2.x) Medium Multiple CVEs (version family) Out of support; patching and app upgrades required.

Evidence & Analysis

  • Why these steps: moving from host reachability to full port coverage reduces blind spots; version banners enable CVE mapping.
  • False positives: validated service presence via repeated scans and header grabs; versions cross-checked with default scripts.
  • Confidence: high for service presence/version; risks corroborated by OpenVAS with CVE references.

Risk Assessment & Remediation Recommendations

  • High priority: Remove or patch vsftpd; upgrade/disable UnrealIRCd; restrict/disable distcc; upgrade MySQL; remove Telnet.
  • Medium priority: Patch Apache/PHP stack; disable unused services; enforce least privilege.
  • Quick wins: Host-based firewall rules (allow-list only), change all default creds, disable guest/anonymous access.
  • Long-term: Regular authenticated scans, configuration baselines (CIS), centralized logging & alerting in SIEM.

Next Steps & Learning Outcomes

  • Follow-ups: authenticated scanning, web application tests, permissioned exploitation for proof-of-impact, SIEM correlation.
  • Skills demonstrated: methodical reconnaissance, Nmap mastery, vulnerability assessment, reporting & communication.
