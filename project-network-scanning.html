Network Scanning & Reconnaissance

End-to-end workflow: host discovery, port/service enumeration, and vulnerability triage with OpenVAS.

Tools: Nmap, OpenVAS Target: Metasploitable2 Attacker: Kali Linux Outcome: Service map + CVE triage

Objective

Map exposed services on the Metasploitable2 target VM and assess vulnerabilities using both manual scans and automated tools.

Outcome

  • Confirmed connectivity between attacker and target VMs.
  • Enumerated open ports, services, and versions with Nmap.
  • Identified vulnerabilities (CVEs) using OpenVAS.
  • Documented remediation recommendations.
View OpenVAS Report Back to Home Lab

Step 1 — Environment & Connectivity

Verified attacker IP address and confirmed reachability to target via ICMP ping.

Step 2 — Full TCP Port Scan

Ran an Nmap full TCP port sweep to enumerate listening services.

Step 3 — Service & OS Detection

Performed version detection and OS fingerprinting to gather banner information and match to CVEs.

Step 4 — Banner Grab & HTTP Headers

Grabbed headers from web service to validate version and confirm attack surface.

Step 5 — Vulnerability Scan (OpenVAS)

Automated vulnerability assessment with OpenVAS. 578 findings, triaged by severity.

Lab Topology

  • Attacker: Kali Linux
  • Target: Metasploitable2
  • SIEM: Ubuntu (Splunk)
  • Network: VirtualBox (Host-only)

Key Skills

Nmap OpenVAS Enumeration Banner Grabbing Vulnerability Management

Severity

Artifacts
