Network Scanning & Reconnaissance

Case study: mapping exposed services, enumerating versions, and assessing vulnerabilities in a controlled lab environment.

Tools: Nmap, curl, OpenVAS Target: Metasploitable2 Attacker: Kali Linux Scope: Service discovery + vulnerability assessment

Project Overview

This project demonstrates the network reconnaissance workflow used in penetration testing and vulnerability management. The objective was to identify all exposed services on a Metasploitable2 target VM, fingerprint versions, and assess vulnerabilities without performing exploitation.

Setup & Environment

Both machines were configured on a 192.168.56.0/24 host-only subnet with no Internet connectivity. This prevents accidental traffic leaks while ensuring bidirectional communication.

Discovery & Reconnaissance

The reconnaissance phase followed a structured methodology:

  1. Host discovery (ICMP, ARP)
  2. Full TCP port scan (-p-)
  3. Service/version detection (-sV)
  4. OS fingerprinting (-O)
  5. Application-layer banner grabbing (curl)
Commands Used
  • nmap -sn 192.168.56.0/24 (ping sweep)
  • nmap -p- 192.168.56.102 -T4 -oN nmap-full.txt (full port scan)
  • nmap -sV -O 192.168.56.102 -oN nmap-services.txt (service & OS detection)
  • curl -I http://192.168.56.102 (HTTP header grab)

Vulnerability Scanning & Reporting

To complement manual scanning, I used OpenVAS (Greenbone Vulnerability Manager). A full "Full and Fast" scan profile was launched against 192.168.56.102.

Commands & Workflow 
# Start scanner services
gvm-start

# Create target
gvm-cli --xml '<create_target><name>Metasploitable2</name><hosts>192.168.56.102</hosts></create_target>'

# Run "Full and Fast" scan
gvm-cli --xml '<create_task>...</create_task>'

# Export report
omp -R --xml="<get_reports report_id="..." format_id="XML"/>" > openvas-report.xml

192.168.56.102 23/tcp Telnet Medium N/A Insecure legacy protocol, susceptible to brute force

Evidence & Analysis

The reconnaissance confirmed multiple legacy services exposed. Key observations:

  • FTP (21/tcp): Running vulnerable vsftpd 2.3.4 (backdoored release).
  • MySQL (3306/tcp): Allowed unauthenticated TCP connections, version outdated.
  • Telnet (23/tcp): Accepts plaintext login, default accounts likely present.
  • HTTP (80/tcp): Apache headers confirm unpatched version.

Risk Assessment & Remediation Recommendations

  • High priority: Remove or patch vsftpd, upgrade MySQL, disable Telnet entirely.
  • Medium priority: Harden Apache config, restrict access to non-essential services.
  • Quick wins: Apply firewall rules to restrict external exposure; enforce SSH with key-based auth only.
  • Long-term: Migrate away from deprecated services, implement regular vulnerability scans.

Next Steps & Learning Outcomes

This project provided hands-on experience with reconnaissance workflows and reinforced the importance of systematic scanning.

  • Next steps: Expand into authenticated scans, integrate results into SIEM, and perform web app testing.
  • Skills demonstrated: Nmap usage, network enumeration, vulnerability scanning, risk triage, technical documentation.
  • Recruiter takeaway: Demonstrates practical security assessment methodology, ability to document findings, and communication of risks/remediations in a clear, professional format.
